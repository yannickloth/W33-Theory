# Collaboration Guidelines 🤝

This file documents simple rules and safe defaults for multiple automated agents and human collaborators working on this repository.

- ✅ Auto-committed artifacts:
  - Generated artifacts (in `committed_artifacts` and `checks`) **may** be auto-committed by the `auto_keep_daemon` and, if enabled, **pushed** to the `auto-keep` branch.
  - The daemon will **not** push to the main/default branch and will avoid overwriting remote changes by attempting to rebase; if that fails it pushes to a timestamped branch instead.

- ✅ Before editing tracked code or artifacts:
  - Run `git fetch origin` and `git status` and `git pull --rebase` to ensure your local copy is up-to-date.
  - When in doubt, create a feature branch (`git checkout -b feat/your-change`) and open a PR rather than pushing directly to `main`.

- ✅ Agent etiquette:
  - Automated agents (e.g., `GitHub Copilot`, `Raptor`, `Claude`) should:
    - Prefer creating a branch and opening a PR for non-trivial code changes.
    - Avoid force-pushing to shared branches.
    - Include an `[agent=NAME]` tag in commit messages for traceability.

- ✅ Auto-keep safety:
  - The `auto_keep_daemon` defaults to **not pushing** (use `--push` to enable).
  - When `--push` and `--branch` are used:
    - The daemon will fetch `origin` and attempt a rebase if the remote has new commits.
    - If rebase fails, the daemon will push to a timestamped branch like `auto-keep-auto-20260207T...` to avoid overwriting others' work.

- ✅ Communication:
  - If you plan to modify artifacts generated by another agent, add a short note to `committed_artifacts/CLAUDE_JOIN.md` (or create a new `committed_artifacts/<AGENT>_TASK.md`) describing your plan and why the change is needed.

---

These guidelines are intentionally light-weight but aim to reduce accidental conflicts and make automated work visible. If you want a stricter workflow (protected branches, required PRs, CI checks), tell us and we can add automation or GitHub settings to enforce it.
